# Domain Migration Guide: polynovearecords.in

This document outlines the steps to migrate from Vercel's auto-generated domain to your custom domain polynovearecords.in.

## Current Setup
- **Website**: polynovea-web.vercel.app (auto-generated by Vercel)
- **Admin**: polynovea-admin-488b.vercel.app (auto-generated by Vercel)
- **Custom Domain**: polynovearecords.in (not yet configured)

## Step 1: Add Custom Domain to Website in Vercel

1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Select the **polynovea-web** project
3. Navigate to **Settings** → **Domains**
4. Click **Add Domain**
5. Enter `polynovearecords.in`
6. Vercel will show you the required DNS records

## Step 2: Configure DNS Records at Your Domain Registrar

Your domain registrar (GoDaddy, Namecheap, etc.) will show where to add DNS records.

Vercel typically requires:
- **Type**: CNAME
- **Name**: `@` (or leave blank for root domain)
- **Value**: `cname.vercel.com` (Vercel will show the exact value)

Or for root domain:
- **Type**: A
- **Value**: `76.76.19.1` (or the IP Vercel provides)

**After adding DNS records, wait 5-30 minutes for propagation.**

## Step 3: Verify Domain in Vercel

Once DNS is configured:
1. Vercel will automatically verify the domain
2. Your website will be accessible at polynovearecords.in
3. Old Vercel URL (polynovea-web.vercel.app) will still work

## Step 4: Update Admin API Access (Later)

Once you have polynovearecords.in configured, you can optionally:

### Option A: Use Admin Subdomain
1. Add `admin.polynovearecords.in` as a custom domain for the polynovea-admin project in Vercel
2. Update `.env.production` to use:
   ```
   NEXT_PUBLIC_ADMIN_API_BASE=https://admin.polynovearecords.in/api/content
   ```

### Option B: Keep Vercel Admin URL
Keep the current setup and only update if needed.

## Environment Variables

The website now uses environment variables for the admin API:

- `.env.local` - Development (current Vercel admin URL)
- `.env.production` - Production (update when admin domain changes)

**Current value**: `https://polynovea-admin-488b.vercel.app/api/content`

Files using this variable:
- `app/blog/page.tsx`
- `components/LivePortfolioExpanded.tsx`

## Rollback

If anything goes wrong:
- DNS can be reverted to the old provider
- Website will fall back to polynovea-web.vercel.app
- No code changes needed (all URLs are in env variables)
